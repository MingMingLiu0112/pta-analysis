## 线段破坏核心逻辑（用户讲解）

### 特征序列分型判断
- 底分型：三根相邻K线端点依次是高高低低高高（中间K线高低都是最低）
- 顶分型：三根相邻K线端点依次是低低高低高低（中间K线高低都是最高）

### 线段破坏两种情形
**常规情形（缺口封闭）**：
- 上行线段回调笔低点 < 前一上行笔高点 → 特征序列缺口封闭
- 反弹上行笔不新高再回落成下行笔，低点 < 前一下行笔低点（特征序列成顶分型）
- → 上行线段在顶分型高点确认结束

**非常规情形（缺口未封闭）**：
- 上行线段回调笔低点 > 前一上行笔高点 → 特征序列缺口未封闭
- 即使反弹回落形成顶分型，**也不能**确认上行线段结束
- 必须等到：该顶分型后的下行线段中，特征序列（上行笔）出现底分型
- → 才能同时确认：上行线段在顶分型结束，下行线段在底分型结束
- 这有点"未来函数"的意思（需要等后续走势确认）

### 关键词
- 缺口封闭 vs 未封闭 → 决定是否立即确认
- 特征序列成顶/底分型 → 确认线段终点的信号

## 晚上进展：线段检测算法突破成功

### 最终算法（简化破坏检查版）
```
核心规则：同方向=延续，反方向=破坏
- DOWN破坏UP：cur.low < prev_opposite.low → 破坏成功
- UP破坏DOWN：cur.high > prev_opposite.high → 破坏成功
- 破坏成功 → 前段结束，新段开始
- 破坏失败 → 前段延续，继续等
```

### 关键教训（用户指正）
- **同方向**高低点抬升/下降 = 延续，不是破坏
- **反方向**才谈得上破坏
- 比较的是笔的**高低点**（bi.low/bi.high），不是端点值
